# 全案营销系统 — 后续 API 接入需求清单

本文档整理了系统各模块在从前端模拟数据切换到真实后端时需要接入的外部 API 和内部接口。

---

## 一、外部第三方 API

### 1. 天眼查 / 企查查 API（工商信息查询）

当前系统中"客户尽调"模块的工商信息查询面板使用的是模拟数据。后续需要接入天眼查或企查查的开放平台 API，以实现以下功能：

| 功能点 | 对应 API | 说明 |
|--------|----------|------|
| 企业基本信息查询 | 企业工商信息接口 | 根据公司名称查询注册资本、法人、成立日期、经营范围等 |
| 股东穿透查询 | 股东信息接口 | 获取股东列表、持股比例、实际控制人 |
| 关联企业查询 | 对外投资 / 分支机构接口 | 查询企业的对外投资和关联公司 |
| 风险信息查询 | 经营异常 / 行政处罚 / 失信被执行 / 司法诉讼接口 | 自动识别企业风险标签 |
| 企业变更记录 | 工商变更接口 | 追踪企业的历史变更信息 |

**预估费用**：天眼查企业版约 5000-20000 元/年，按查询次数计费。企查查类似。

### 2. 电子签章 API（合同签署）

合同签约模块目前的签署流程为模拟操作。后续需要接入电子签章服务以实现法律效力的电子合同签署。

| 功能点 | 推荐服务商 | 说明 |
|--------|-----------|------|
| 合同模板管理 | e签宝 / 法大大 / 上上签 | 预设合同模板，自动填充客户和报价信息 |
| 电子签名 | e签宝 / 法大大 | 支持手写签名、CA 证书签名 |
| 合同归档与查验 | e签宝 / 法大大 | 签署完成后自动归档，支持合同真伪查验 |

**预估费用**：e签宝按份计费，约 5-15 元/份；年度套餐约 3000-10000 元。

### 3. 短信/邮件通知 API

审批流程和合同签署需要通知相关人员。

| 功能点 | 推荐服务商 | 说明 |
|--------|-----------|------|
| 审批通知 | 阿里云短信 / 腾讯云短信 | 审批发起、通过、驳回时通知相关人 |
| 合同签署提醒 | 同上 + SendGrid (邮件) | 提醒客户签署合同 |
| 风险预警推送 | 同上 | 客户关联企业出现新增风险时推送通知 |

**预估费用**：短信约 0.04-0.06 元/条；邮件服务约 100-500 元/月。

## 二、内部后端 API（需升级为全栈项目）

当前系统为纯前端静态项目，所有数据存储在前端模拟数据中。升级为全栈项目后需要实现以下后端接口：

### 1. 认证与权限

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/auth/login` | POST | 用户登录，返回 JWT Token |
| `/api/auth/register` | POST | 用户注册 |
| `/api/auth/logout` | POST | 用户登出 |
| `/api/users` | GET/POST/PUT/DELETE | 用户管理 CRUD |
| `/api/roles` | GET | 获取角色列表和权限配置 |

### 2. 商机管理

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/opportunities` | GET/POST | 商机列表和创建 |
| `/api/opportunities/:id` | GET/PUT/DELETE | 商机详情、更新、删除 |
| `/api/opportunities/:id/stage` | PUT | 推进商机阶段 |
| `/api/opportunities/:id/follow-ups` | GET/POST | 跟进记录 |

### 3. 客户尽调

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/due-diligence/:oppId` | GET/PUT | 尽调数据读写 |
| `/api/due-diligence/:oppId/business-info` | GET | 工商信息查询（对接天眼查） |
| `/api/due-diligence/:oppId/risk-scan` | GET | 风险扫描（对接天眼查） |

### 4. 行业诊断

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/diagnoses/:oppId` | GET/PUT | 诊断数据读写 |
| `/api/diagnoses/:oppId/report` | GET | 生成诊断报告 |

### 5. 全案方案

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/proposals/:oppId` | GET/PUT | 方案数据读写 |
| `/api/proposals/:oppId/versions` | GET/POST | 方案版本管理 |
| `/api/proposals/:oppId/feasibility` | GET | 可行性校验 |

### 6. 报价管理

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/quotations` | GET | 报价列表 |
| `/api/quotations/:oppId` | GET/PUT | 报价详情和编辑 |
| `/api/quotations/:oppId/calculate` | POST | 价格计算引擎 |

### 7. 审批中心

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/approvals` | GET | 审批列表（按角色过滤） |
| `/api/approvals/:id/approve` | POST | 通过审批 |
| `/api/approvals/:id/reject` | POST | 驳回审批 |

### 8. 合同签约

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/contracts/:oppId` | GET/PUT | 合同数据读写 |
| `/api/contracts/:oppId/sign` | POST | 发起签署（对接电子签章） |
| `/api/contracts/:oppId/pdf` | GET | 生成合同 PDF |

---

## 三、接入优先级建议

| 优先级 | API | 理由 |
|--------|-----|------|
| P0 - 必须 | 内部后端 API（全栈升级） | 数据持久化是系统上线的基础 |
| P0 - 必须 | 天眼查/企查查 API | 工商信息查询是尽调模块的核心价值 |
| P1 - 重要 | 短信/邮件通知 API | 审批和签约流程需要及时通知 |
| P2 - 增强 | 电子签章 API | 提升合同签署的法律效力和效率 |
