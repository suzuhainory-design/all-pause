# 全链路 AI 视频智能剪辑与生成平台

> 从用户需求、素材入库、脚本反推、智能剪辑、BGM 匹配，到巨量千川检测和成片保存，一键完成短视频生产闭环。

## 项目简介

本项目是一款面向内容创作者、短视频运营团队、品牌投放团队和剪辑师的端到端 AI 视频生产工作台。系统通过智能体驱动，把用户的模糊需求逐步澄清为可执行的视频生产任务，并自动完成素材入库、素材补齐、优质视频脚本反推、脚本归档、分镜选材、BGM 匹配、视频合成、巨量千川首发/优质检测和成片保存。

平台的核心目标不是只生成一条视频，而是沉淀可复用的素材库、脚本库和成片资产，让后续同类视频可以更快复刻、更稳定投放。

## 核心功能

| 功能模块 | 说明 | 核心价值 |
|---|---|---|
| **用户登入界面** | 进入独立工作空间，绑定项目与平台 | 隔离数据，个性化配置 |
| **素材工作流** | 用户上传素材后精准入库；素材库缺失时支持智能体联网查找或生成视频补齐 | 建立结构化素材资产，解决素材荒 |
| **脚本工作流** | 从网络优质视频或用户上传的优质视频反推脚本，并保存到脚本库 | 沉淀爆款逻辑，实现结构复用 |
| **脚本双类型库** | 脚本分为「口播类型」和「情景剧类型」，便于后续检索、复刻和改写 | 针对不同内容形态精细化管理 |
| **剪辑工作流** | 根据脚本内容、分镜结构、情绪节奏自动选择合适素材并生成剪辑计划 | 自动化剪辑，提升生产效率 |
| **BGM 选择工作流** | 根据视频素材时长 + 转场时长，在开源音乐素材库中匹配合适 BGM | 智能配乐，增强视频感染力 |
| **一键成片工作流** | 用户选择视频分类后，智能体继续追问缺失信息，让需求从模糊变具体 | 降低使用门槛，引导式创作 |
| **脚本上传/选择** | 支持用户上传脚本或从脚本库选择；上传脚本会按脚本案例结构化处理 | 灵活接入已有创意 |
| **巨量千川接口检测** | 对成片进行首发检测和优质检测，辅助判断是否适合投放 | 提前规避投放风险，提高过审率 |
| **成片保存** | 支持用户保存成片、剪映草稿和相关生产记录 | 资产沉淀，方便二次编辑与复盘 |

## 产品主流程

```text
用户登入界面
    ↓
素材工作流 (上传/解析/打标/入库)
    ↓
脚本工作流 (反推/拆解/归档)
    ↓
剪辑工作流 (分镜拆解/素材匹配/剪辑计划)
    ↓
BGM 选择工作流 (时长计算/情绪匹配)
    ↓
用户一键成片工作流 (需求澄清/任务生成)
    ↓
巨量千川接口检测 (首发/优质/风险评估)
    ↓
保存成片 (视频/草稿/记录归档)
```

### 1. 用户登入界面

用户进入系统后，先完成登入并进入自己的工作空间。登入界面主要承担三个作用：

- **绑定项目空间**：后续素材库、脚本库、BGM 库、成片记录都归属到当前用户或项目。
- **承接生产入口**：登入后直接进入「一键成片」工作台，也可以单独进入素材库、脚本库、BGM 库和草稿中心。
- **个性化配置**：记录用户的偏好设置，如默认平台、默认视频类型等。

建议后续扩展字段：

| 字段 | 说明 | 示例 |
|---|---|---|
| 项目名称 | 当前视频项目或投放项目 | 2024夏日防晒霜推广 |
| 默认平台 | 抖音 / 小红书 / 视频号 / B 站等 | 抖音 |
| 默认视频类型 | 种草、口播、剧情、探店、品牌宣传等 | 种草带货 |
| 偏好风格 | 视频整体的视觉和听觉风格 | 活泼、专业、极简 |

### 2. 素材工作流

素材工作流负责把用户上传的素材变成可被智能体理解和调用的结构化素材库。

核心流程：

```text
用户上传素材 / 扫描本地文件夹
    ↓
素材格式校验 (支持 mp4, mov 等常见格式)
    ↓
视频基础信息解析：时长、分辨率、帧率、文件大小、编码格式
    ↓
质量检测：清晰度、稳定性、是否抖动、是否过短、是否存在黑边
    ↓
抽帧 / 缩略图生成 (用于前端展示和 AI 视觉分析)
    ↓
AI 识别内容：场景、人物、物体、情绪、镜头运动、适用场景、画面色彩
    ↓
写入素材库 (数据库存储元数据，文件系统存储实体文件)
```

素材入库后，每条素材建议包含以下信息：

| 字段 | 说明 | 示例 |
|---|---|---|
| 素材 ID | 系统生成的唯一标识 | mat_20240501_001 |
| 文件路径 | 原始视频或处理后视频路径 | /data/materials/video_001.mp4 |
| 文件名称 | 用户上传的原始名称 | 户外跑步_原素材.mp4 |
| 时长 | 视频素材秒数 | 15.5s |
| 分辨率 / 帧率 | 用于判断是否适合目标平台 | 1080x1920 / 30fps |
| 质量评分 | 判断是否适合进入剪辑流程 (0-100) | 85 |
| 稳定片段 | 如果素材抖动，记录可用的稳定区间 | [2.0s, 10.5s] |
| 场景标签 | 如室内、户外、产品特写、人物口播 | 户外, 公园 |
| 情绪标签 | 如轻松、专业、紧张、温馨、激情 | 轻松, 活力 |
| 主体标签 | 如人物、产品、店铺、食物、风景 | 青年女性, 运动鞋 |
| 适用分镜 | 如开场、卖点展示、情绪转折、结尾 CTA | 卖点展示, 情绪渲染 |
| 画面色彩 | 主色调，用于视觉统一性匹配 | 暖色调, 绿色为主 |

当素材库没有合适素材时，用户可以选择两种补齐方式：

- **智能体联网查找**：根据分镜需求、关键词和风格，在网络或开放素材源中查找合适素材。
- **生成视频素材**：根据缺失分镜自动生成短片段，例如产品特写、场景氛围、转场画面等。

素材补齐策略建议：

| 策略 | 使用场景 | 优点 | 注意事项 |
|---|---|---|---|
| 素材库优先 | 用户已有大量素材 | 风格统一、成本低、可控性强 | 可能缺少特定镜头，需定期维护 |
| 联网查找 | 需要通用场景或补充镜头 | 快速补齐，资源丰富 | 需注意版权和授权，可能存在质量参差不齐 |
| 生成视频 | 需要高度定制画面，且无现成素材 | 与脚本强匹配，创意无限 | 成本和生成时间更高，可能存在 AI 痕迹 |

### 3. 脚本工作流

脚本工作流负责把优质内容的结构沉淀下来，让系统不仅能生成脚本，还能复用爆款结构。

脚本来源分为三类：

| 来源 | 说明 | 适用场景 |
|---|---|---|
| 网络优质视频反推 | 用户提供优质视频链接或参考内容，智能体反推脚本结构 | 竞品分析，爆款拆解 |
| 用户上传优质视频反推 | 用户上传视频后，系统判断为优质，再提取并保存脚本 | 内部优秀案例沉淀 |
| 用户手动上传脚本 | 用户已有脚本文案，系统按案例结构拆解并补齐字段 | 已有创意落地 |

脚本库必须分为两大类：

#### 口播类型

适用于知识讲解、产品种草、观点输出、品牌介绍、课程引流等视频。

典型结构：

```text
开场钩子 (黄金3秒，抓住眼球)
    ↓
痛点 / 场景共鸣 (引发共鸣，建立连接)
    ↓
核心观点 / 产品卖点 (提供解决方案)
    ↓
案例或证明 (增强信任感)
    ↓
行动引导 / CTA (促成转化)
```

建议记录字段：

| 字段 | 说明 | 示例 |
|---|---|---|
| 开场钩子 | 前 3 秒吸引注意力的话术 | "你还在每天熬夜加班吗？" |
| 目标人群 | 面向谁说 | 职场新人, 熬夜党 |
| 核心痛点 | 用户为什么需要看下去 | 效率低下, 身体透支 |
| 主要卖点 | 产品或观点的核心价值 | 提升效率的 3 个神器 |
| 语气风格 | 专业、轻松、犀利、温情等 | 犀利, 干货 |
| CTA | 关注、咨询、购买、预约等 | "点击左下角链接获取完整清单" |

#### 情景剧类型

适用于剧情短剧、反转故事、冲突对话、品牌植入、本地生活探店等视频。

典型结构：

```text
人物设定 (明确角色关系)
    ↓
场景建立 (交代故事背景)
    ↓
冲突出现 (制造矛盾，推动剧情)
    ↓
反转 / 爽点 (意料之外，情理之中)
    ↓
产品或主题露出 (自然植入)
    ↓
结尾记忆点 (强化印象)
```

建议记录字段：

| 字段 | 说明 | 示例 |
|---|---|---|
| 人物角色 | 主角、配角、关系设定 | 老板 (严厉), 员工 (机智) |
| 场景设定 | 办公室、家中、门店、街头等 | 办公室会议室 |
| 冲突点 | 推动剧情发展的矛盾 | 老板要求半小时内完成不可能的任务 |
| 反转点 | 观众意料之外的变化 | 员工用 AI 工具 5 分钟搞定 |
| 台词 | 每个角色的具体表达 | (详见脚本正文) |
| 分镜 | 每一幕对应的画面描述 | (详见分镜列表) |
| 植入方式 | 产品、品牌、服务如何自然出现 | 员工展示电脑屏幕上的 AI 工具界面 |

脚本入库后，支持：

- 按类型筛选：口播类型 / 情景剧类型。
- 按标签筛选：行业、平台、风格、情绪、产品类型。
- 按案例复刻：选择脚本 ID，输入新主题，生成同结构新脚本。
- 按缺失字段追问：用户上传脚本信息不完整时，智能体列出需要补充的问题。

### 4. 剪辑工作流

剪辑工作流负责把脚本变成可执行的剪辑计划，并从素材库中自动选择合适素材。

核心流程：

```text
读取脚本 (获取文本和结构信息)
    ↓
拆解脚本结构 (划分为独立的分镜单元)
    ↓
生成分镜列表 (明确每个画面的具体需求)
    ↓
为每个分镜提取素材需求 (标签化需求：场景、人物、动作等)
    ↓
从素材库检索匹配素材 (基于向量检索或标签匹配)
    ↓
缺失素材触发联网查找或视频生成 (补齐素材缺口)
    ↓
生成剪辑计划：素材顺序、时长、转场、字幕、节奏、特效
    ↓
FFmpeg 合成视频 / 导出剪映草稿 (生成最终产物)
```

每个分镜建议包含：

| 字段 | 说明 | 示例 |
|---|---|---|
| 分镜序号 | 当前片段顺序 | 01 |
| 脚本文案 | 当前分镜对应台词或字幕 | "夏天到了，你还在为防晒发愁吗？" |
| 画面描述 | 需要什么画面 | 阳光强烈的海滩，人物皱眉遮挡阳光 |
| 素材标签 | 场景、人物、主体、情绪、运动方式 | 海滩, 阳光, 烦恼, 遮挡 |
| 预计时长 | 该分镜持续秒数 | 3.0s |
| 转场建议 | 切、淡入淡出、推拉、闪白等 | 快速推镜头 |
| 字幕样式 | 标题、底部字幕、强调字等 | 底部大字，带黄色描边 |
| 素材来源 | 素材库 / 联网查找 / AI 生成 | 素材库 (mat_005) |
| 特效建议 | 滤镜、贴纸、音效等 | 增加阳光刺眼的滤镜，配上"滋"的音效 |

素材选择优先级建议：

1. 与分镜标签完全匹配的高质量素材。
2. 与情绪、场景、主体部分匹配的素材。
3. 同类型脚本历史使用效果较好的素材。
4. 联网查找素材。
5. AI 生成补齐素材。

### 5. BGM 选择工作流

BGM 工作流负责根据视频整体节奏、素材时长和转场时长选择合适音乐。

匹配逻辑：

```text
统计素材总时长 (所有分镜时长之和)
    ↓
统计转场总时长 (所有转场效果耗时之和)
    ↓
计算成片预计总时长 (素材总时长 + 转场总时长)
    ↓
分析脚本情绪和视频节奏 (提取情绪关键词，判断节奏快慢)
    ↓
检索开源音乐素材库 (基于时长、情绪、BPM 过滤)
    ↓
按时长、BPM、情绪、风格打分 (综合评估匹配度)
    ↓
返回最佳 BGM 和候选列表 (供系统自动应用或用户手动选择)
```

推荐 BGM 时需要考虑：

| 维度 | 说明 | 示例 |
|---|---|---|
| 总时长 | BGM 时长需要覆盖视频素材时长 + 转场时长 | 视频 35s，BGM 需 >= 35s |
| BPM | 快节奏视频适合更高 BPM，情绪型视频适合中低 BPM | 动感混剪 (120+ BPM)，抒情口播 (60-80 BPM) |
| 情绪 | 与视频内容情绪一致，如轻快、温情、紧张、专业 | 搞笑剧情配滑稽音效，专业科普配沉稳背景音 |
| 平台风格 | 抖音、小红书、视频号等平台节奏不同 | 抖音偏好强节奏、洗脑旋律 |
| 版权 | 优先使用开源或明确授权的音乐素材 | CC0 协议音乐，或已购买商用版权的曲库 |

### 6. 用户一键成片工作流

一键成片工作流是用户侧的主入口。用户不需要一次性填写完整需求，只需要先选择分类，智能体会继续追问，把模糊需求变成可执行任务。

用户输入方式：

| 输入项 | 说明 | 示例 |
|---|---|---|
| 视频分类 | 种草带货、知识口播、剧情短剧、探店本地生活、品牌宣传等 | 种草带货 |
| 创作意图 | 用户用自然语言描述想做什么视频 | "推销一款新型扫地机器人，强调吸力大和免洗拖布" |
| 目标平台 | 抖音、小红书、视频号、B 站等 | 抖音 |
| 视频时长 | 15 秒、30 秒、60 秒或自定义 | 30 秒 |
| 脚本来源 | 用户上传、脚本库选择、智能体生成/反推 | 智能体生成 |
| 素材来源 | 上传素材、素材库、联网查找、生成视频 | 素材库 + 生成视频 |
| BGM 偏好 | 轻快、专业、温情、热血、悬疑等 | 科技感, 节奏明快 |

智能体追问规则：

- 如果用户需求中缺少目标人群，需要追问「这条视频主要给谁看？」
- 如果缺少产品卖点，需要追问「最想突出的 1-3 个卖点是什么？」
- 如果脚本台词模糊，需要追问「是否需要更强口播、剧情化表达还是字幕驱动？」
- 如果素材不足，需要追问「缺失素材是允许联网查找，还是生成视频补齐？」
- 如果成片用于投放，需要追问「是否需要通过巨量千川首发和优质检测？」

示例：种草带货分类下的追问

```text
1. 产品名称和核心卖点是什么？
2. 目标人群是谁？
3. 是否有价格、优惠、购买入口或口播 CTA？
4. 风格更偏真实测评、达人种草，还是品牌广告？
5. 缺失素材时，优先联网查找还是生成视频？
```

示例：情景剧分类下的追问

```text
1. 主角和配角是谁？
2. 故事发生在什么场景？
3. 冲突点是什么？
4. 反转或爽点是什么？
5. 产品或品牌要在第几幕出现？
```

### 7. 巨量千川接口检测

成片完成后，系统可对视频进行巨量千川相关检测，辅助判断视频是否适合后续投放。

检测目标：

| 检测项 | 说明 | 价值 |
|---|---|---|
| 首发检测 | 判断素材或成片是否满足首发要求 | 争取平台首发流量扶持 |
| 优质检测 | 判断视频内容是否达到优质素材标准 | 提高跑量概率，降低转化成本 |
| 低质风险 | 检测画面质量、重复、搬运、低效素材等风险 | 避免账号降权或封禁 |
| 投放建议 | 返回是否建议进入投放、是否需要修改 | 提供决策依据，优化投放策略 |

检测流程：

```text
成片生成 (获取最终 mp4 文件)
    ↓
上传检测素材 / 调用巨量千川接口 (通过 API 提交检测任务)
    ↓
获取检测任务 ID (异步处理)
    ↓
轮询检测结果 (定时查询任务状态)
    ↓
解析首发、优质、低效、风险原因 (结构化返回数据)
    ↓
生成检测报告 (可视化展示检测结果)
    ↓
决定通过、拦截或建议修改 (系统自动判定或人工审核)
```

检测结果建议分为三类：

| 状态 | 说明 | 后续动作 |
|---|---|---|
| 通过 | 首发和优质检测均满足要求 | 可以保存并进入投放准备，一键推送到投放平台 |
| 待优化 | 存在部分问题但可修改 (如字幕遮挡、部分画面模糊) | 返回修改建议，重新剪辑或替换素材后再次检测 |
| 拦截 | 不满足首发或优质要求 (如严重违规、高度重复) | 阻止保存为投放成片，要求重新生成或废弃 |

### 8. 保存成片

保存功能用于把最终视频、草稿和生产过程记录沉淀下来，方便复用、审核和二次编辑。

建议保存内容：

| 内容 | 说明 | 格式/存储方式 |
|---|---|---|
| 成片视频 | 最终合成的视频文件 | `.mp4`，存储于云端或本地文件系统 |
| 剪映草稿 | 可二次编辑的工程文件 | `.draft`，包含轨道、素材、特效等信息 |
| 使用脚本 | 本次成片使用的脚本正文和脚本类型 | JSON 格式，关联脚本库 ID |
| 使用素材 | 每个分镜使用的素材 ID、来源和时长 | JSON 格式，关联素材库 ID |
| BGM 信息 | 使用的音乐名称、来源、授权信息 | JSON 格式，关联 BGM 库 ID |
| 检测报告 | 巨量千川首发/优质检测结果 | PDF 或 HTML 报告 |
| 生成参数 | 平台、时长、分类、风格、用户需求等 | JSON 格式，记录完整 Prompt 和配置 |
| 保存时间 | 成片保存时间和项目归属 | Timestamp，关联项目 ID |

保存后的成片可以进入：

- 草稿中心：下载剪映草稿，人工二次优化，微调细节。
- 成片库：查看、预览、下载、复用，作为后续创作的参考。
- 投放准备：查看检测结果，确认是否进入投放，对接广告投放系统。
- 复盘分析：对比不同脚本、素材、BGM 的成片效果，优化生产策略。

## 使用指南

### 环境准备

1. 安装 Python 3.9+
2. 安装 FFmpeg 并配置环境变量
3. 安装依赖：`pip install -r requirements.txt`
4. 复制 `config/config.example.yaml` 为 `config/config.yaml` 并填写必要的 API Key (如 OpenAI, 巨量千川等)

### 启动服务

```bash
# Windows
scripts\start.bat

# Linux/Mac
bash scripts/start.sh
```
服务启动后，访问 `http://localhost:8000` 进入工作台界面。

### 一键工作流

1. 在「一键工作流」页面输入创作意图，例如：
   > 帮我做一条30秒的夏日防晒霜种草视频，风格轻快活泼，适合抖音投放
2. 选择目标平台、视频时长和生成渠道
3. 可选：上传本地素材（不上传则 AI 自动生成或联网查找）
4. 点击「⚡ 一键生成」，实时查看执行进度 (素材处理 -> 脚本生成 -> 剪辑合成)
5. 完成后在「草稿中心」下载剪映草稿文件，或直接预览成片

### 脚本库管理

- **生成脚本**：输入主题直接生成新脚本，自动结构化。
- **拆解入库**：粘贴优质脚本或视频链接，AI 提取逻辑框架并保存。
- **改写复刻**：选择参考脚本 ID，输入新主题，一键改写，保留爆款逻辑。

### BGM 库扩充

编辑 `resource/bgms/meta.json`，按以下格式添加音乐文件：

```json
{
  "id": "bgm_001",
  "name": "夏日元气轻快",
  "file": "summer_pop.mp3",
  "bpm": 120,
  "duration": 180,
  "mood": "欢快活泼",
  "scene": "户外, 种草, 探店",
  "tags": ["流行", "电子", "轻快"]
}
```

将对应的 `.mp3` 文件放入 `resource/bgms/` 目录。系统会自动加载并用于后续的智能匹配。

## 技术架构

```text
[前端界面 (Vue/React)] <---> [FastAPI 后端接口]
                                  |
                                  v
                          [VideoAgent (LLM 意图解析与任务规划)]
                                  |
                                  v
                          [VideoPipeline (工作流编排执行)]
                                  |
    +-----------------------------+-----------------------------+
    |                             |                             |
[素材处理模块]                [脚本处理模块]                [剪辑合成模块]
- 视频解析 (OpenCV/FFmpeg)    - 结构反推 (LLM)              - 分镜匹配 (向量检索)
- 智能打标 (Vision Model)     - 逻辑改写 (LLM)              - 剪映草稿生成 (JSON 组装)
- 联网检索 (Search API)       - 结构化存储 (DB)             - 视频渲染 (FFmpeg)
- 视频生成 (Seedance API)                                   - BGM 匹配 (音频特征分析)
```

## 演进路线

- **Phase 1（当前）**：基础工作流跑通，Seedance 接入，大模型打标，实现从文本到视频的基础闭环。
- **Phase 2**：脚本库完善（支持更多细分类型），BGM 精准匹配（基于音频波形和情绪曲线），剪映草稿稳定导出（支持更多特效和转场）。
- **Phase 3**：本地打标小模型部署（降低 API 成本，提升处理速度），多渠道扩展（支持更多视频生成大模型），数据回流优化（根据投放效果反哺脚本和素材选择策略）。
- **Phase 4**：多智能体协作（引入导演 Agent、剪辑 Agent、审核 Agent，实现更复杂的视频生产逻辑），支持批量矩阵化生产。
